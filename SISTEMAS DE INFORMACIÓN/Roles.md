# ANÁLISIS DE LOS ROLES Y PARTICIPANTES EN EL DESARROLLO DE SOFTWARE #

Concepto importante:  
`Stakeholders: Partes Interesadas`  

- Incluyendo tanto a los desarrolladores, usuarios y clientes.
- La importancia de identificarlos esta en que nos permite recopilar requisitos para el sistema, nos ayuda a reducir conflictos y mayor aceptación.
---
 ### CATEGORÍA FUNCIONAL 
 - Usuarios Operativos  
   Aquellos que interactuan directamente y ejecutan tareas rutinarias.
   
 - Usuarios administrativos / tacticos  
   Supervisan procesos, requieren reportes y toman decisiones a mediano plazo.
   
 - Usuarios estratégicos  
   Alta gerencia que requiere información resumida y analítica para apoyar decisiones estratégicas.
   
 - Usuarios externos  
   Clientes, proveedores y entidades reguladoras.
---   
 ### POR NIVEL DE EXPERIENCIA TECNOLÓGICA 
  - Usuarios Novatos  
    Poco conocimiento informático , requieren ayuda de una interfaz simple.
    
  - Usuarios Intermedios  
    Manejan funciones básicas y algunas avanzadas
    
  - Usuarios Expertos  
    Dominan el sistema y buscan eficiencia y personalización
---
### ROLES
  - ANALISTA DE SISTEMAS  
    Traduce las necesidades del cliente en requisitos claros y estructurados.  
    Sirve de puente entre ussuarios y equipo técnico, garantiza que el sistema cumpla los objetivos.
    
```
  FUNCIONES PRINCIPALES
  - Levantamiento y análisis de requisitos
  - Documentación técnicca y funcional
  - Modelado de procesos (UML, casos de uso)
  - Validación con el cliente
```

  - DESARROLLADOR  
    Codifica y desarrolla la aplicación con herramientas de programación  
    
  - TESTER  
    Registra defectos y valida el cumplimiento previo a la entrega.  
    Busca fallas o bugs.  
    
  - JEFE DE PROYECTO  
    Planifica y coordina sobre el alcance, tiempo, costo y recursos
    Tiene responsabilidades
